# 回测判断标准定义
# ============================================================

## 1X2 (胜平负) 判断标准
- 模型预测最可能结果 vs 实际结果
- 如果推荐"双选"（如摩洛哥不败），只要命中一个即算正确
- 如果推荐"避开"（如巴西胜为负价值），实际巴西未胜则推荐正确

## 2. 让球胜平负 (Asian Handicap) 判断标准
- 根据推荐盘口和实际比分计算让球结果
- 判断标准：推荐方向是否赢盘
- 例如：推荐摩洛哥+1.5，实际1-1，则摩洛哥赢盘✅

## 3. 半全场 (HT/FT) 判断标准
- 预测最可能组合 vs 实际半全场结果
- 注：需要实际半场数据，但已结束比赛目前只有全场比分
- 暂以预测最可能组合是否正确标记

## 4. 比分 (Correct Score) 判断标准
- 预测最可能比分 vs 实际比分
- 如果推荐包含多个比分，命中任一即算正确

## 5. 进球数 (Total Goals) 判断标准
- 预测最可能总进球数 vs 实际总进球数
- 区间预测：如果预测"1-2球"，实际1或2球都算正确

## 6. 大小球 (Over/Under) 判断标准【重点修正】
- 核心问题：不同盘口有不同规则，必须明确

### 大小球盘口规则定义：

**2.0球盘口**：
- 0-1球 = 小球赢全
- 2球 = 走水（退本金）
- 3球+ = 大球赢全
- 判断标准：推荐Over且≥3球✅，推荐Under且≤1球✅，2球=走水➖

**2.25球盘口（2/2.5或2-2.5）**：
- 0-1球 = 小球赢全
- 2球 = 小球赢一半，大球输一半
- 3球+ = 大球赢全
- 判断标准：推荐Over且≥3球✅，推荐Under且≤1球✅，2球=小球赢半✅

**2.5球盘口**：
- 0-2球 = 小球赢全
- 3球+ = 大球赢全
- 无走水
- 判断标准：推荐Over且≥3球✅，推荐Under且≤2球✅

**2.75球盘口（2.5/3）**：
- 0-2球 = 小球赢全
- 3球 = 大球赢一半，小球输一半
- 4球+ = 大球赢全
- 判断标准：推荐Over且≥4球✅，推荐Under且≤2球✅，3球=大球赢半✅

**4/4.5球盘口**：
- 0-4球 = 小球赢全（或4球赢半）
- 5球+ = 大球赢全
- 判断标准：根据具体规则

### 模型大小球判断逻辑错误分析：

**错误#1：盘口混淆**
- 巴西vs摩洛哥报告：推荐"小球2.25"但盘口列了2.0、2.25、2.5、2.5/3多个盘口
- 实际判断时用了"小球58%"但未明确基于哪个盘口
- 实际结果1-1（2球），在2.25盘口下小球赢半，但报告写的是"小球赢"

**错误#2：推荐与盘口不一致**
- 瑞典vs突尼斯报告：盘口是2/2.5（2.25），但推荐写的是"小球2.5"
- 2.25和2.5是不同盘口！2.25下2球=小球赢半，2.5下2球=小球赢全
- 模型推荐"小球2.5"但用2.25盘口计算概率，逻辑不一致

**错误#3：Edge计算未考虑走水**
- 2.0盘口下2球=走水，但模型计算概率时未区分
- 2.25盘口下2球=小球赢半，但模型只计算"小球赢全"概率
- 导致Edge计算偏高（尤其是小球推荐）

**错误#4：总进球预期 vs 大小球推荐矛盾**
- 巴西vs摩洛哥：总进球预期1-2球（54%），但大小球推荐2.25小球（58%）
- 预期总进球2.0球，2.25盘口下2球=小球赢半，但模型按赢全计算
- 科特迪瓦vs厄瓜多尔：预期总进球2.0-2.3球，推荐2.0大球（微弱edge）
- 2.0盘口下2球=走水，但模型预测"大球55%"未考虑走水概率

# 大小球逻辑错误总结（重点）
# ============================================================

## 发现的4类大小球逻辑错误：

### 错误类型A：盘口规则理解错误（最严重）
**案例：瑞典vs突尼斯**
- 2/2.5（2.25）盘口下，报告说"2球=小球赢全"
- 正确规则：2球=小球赢半
- 影响：模型按赢全计算概率，导致小球概率虚高（61%→实际应考虑赢半约55%）
- 后果：推荐小球，实际4球大球，严重错误

### 错误类型B：推荐与盘口不一致
**案例：瑞典vs突尼斯**
- 盘口是2/2.5（2.25），但推荐写"小球2.5"
- 2.25和2.5是不同盘口！
- 2.25下2球=小球赢半，2.5下2球=小球赢全
- 模型混淆盘口，导致概率计算错误

**案例：巴西vs摩洛哥**
- 列出多个盘口（2.0、2.25、2.5、2.5/3），推荐"小球2.25"
- 但未明确基于哪个盘口计算概率
- 2.25下58% vs 2.5下可能只有50%

### 错误类型C：走水机制忽略
**案例：科特迪瓦vs厄瓜多尔**
- 2.0盘口下，推荐大球2+，但预测最可能进球数=2球（走水）
- 模型未区分"2球=走水"和"3球+=大球赢"
- Edge计算时把2球也算作大球赢，导致大球概率虚高（55%）
- 实际1球=小球赢全，与推荐相反

**案例：德国vs库拉索**
- 4/4.5盘口下，报告说"走水（正好4球）概率高"
- 实际8球=大球赢全，走水概率一点也不高
- 模型对强弱悬殊比赛走水概率严重高估

### 错误类型D：同一报告内市场矛盾
**案例：荷兰vs日本**
- 比分推荐：2:2（4球）
- 大小球推荐：小球2.5（4球=大球）
- 矛盾原因：冷门雷达修正了1X2方向，但未联动修正大小球
- 日本在冷门情况下进攻爆发，模型未考虑

**案例：巴西vs摩洛哥**
- 总进球预期：1-2球（54%）
- 大小球推荐：2.25小球（58%）
- 矛盾：如果预期1-2球，2.25盘口下2球=小球赢半，不应按赢全计算58%

# 改进建议（针对大小球逻辑）
# ============================================================

## P0 - 立即修复
1. **修正2.25盘口规则**：2球=小球赢半，不是赢全
2. **修正2.0盘口规则**：2球=走水，不是大球赢
3. **统一推荐与盘口**：推荐必须明确对应盘口（不能推荐2.5但用2.25计算）
4. **大小球概率计算**：
   - 2.0盘口：大球=≥3球，小球=≤1球，2球=走水（概率单独列出）
   - 2.25盘口：大球=≥3球，小球=≤1球，2球=小球赢半
   - 2.5盘口：大球=≥3球，小球=≤2球，无走水
   - 2.75盘口：大球=≥4球，小球=≤2球，3球=大球赢半

## P1 - 模型改进
5. **大小球与比分联动**：如果比分推荐2:2（4球），大小球必须推荐大球
6. **冷门因子联动修正**：冷门雷达修正1X2时，必须同时修正大小球（冷门球队可能进攻爆发）
7. **强弱悬殊比赛**：强弱差>3球时，大小球盘口应自动上调（4.5/5.0）
8. **进球数分布校准**：当前模型对低比分概率高估，对高比分概率低估

## P2 - 系统优化
9. **大小球Edge计算**：必须考虑走水和赢半机制
   - 2.25盘口Edge = (P≤1球)×1 + (P=2球)×0.5 - 1/(赔率)
   - 不能简单用 P(≤2球) vs 市场隐含概率
10. **多盘口对比**：报告应列出推荐盘口及对应概率，避免混淆
